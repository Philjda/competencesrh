Cas concret : utiliser le machine learning pour réduire le turnover en entreprise

// Publié le 20 mai 2026 · 12 min de lecture

Pourquoi les DRH utilisent désormais l’IA prédictive

Le turnover représente aujourd’hui un enjeu stratégique majeur pour les entreprises. La perte de profils techniques, l’augmentation des coûts de recrutement et la difficulté à fidéliser les collaborateurs poussent de plus en plus de DRH à utiliser la data RH et l’intelligence artificielle.

Grâce au People Analytics, il devient possible de détecter les signaux faibles avant les départs et d’agir de manière proactive.

Contexte de la PME

L’entreprise fictive TechFab Solutions évolue dans le secteur de l’industrie et de la digitalisation industrielle.

  • 180 salariés ;
  • forte tension sur les profils techniques ;
  • turnover passé de 9 % à 18 % en deux ans ;
  • difficultés de recrutement ;
  • augmentation des démissions.

Le DRH pose alors une question simple :

“Peut-on prédire quels salariés risquent de quitter l’entreprise dans les prochains mois ?”

Objectif du projet IA RH

L’objectif du projet est de construire un modèle prédictif capable de :

  • détecter les salariés à risque ;
  • identifier les causes principales ;
  • aider les managers à agir avant les départs.

Étape 1 : compréhension métier RH

Avant de construire un modèle IA, le consultant RH commence toujours par une analyse métier.

Plusieurs hypothèses émergent :

  • salaire insuffisant ;
  • surcharge de travail ;
  • faible reconnaissance ;
  • manque d’évolution ;
  • management défaillant ;
  • temps de trajet élevé ;
  • désengagement collaborateur.

Étape 2 : collecte des données RH

Les données sont récupérées depuis le SIRH et les enquêtes internes.

Variable Description
Age Âge du salarié
Anciennete Nombre d’années dans l’entreprise
Salaire Rémunération annuelle
HeuresSup Heures supplémentaires
Distance Distance domicile-travail
Satisfaction Score enquête interne
Formation Heures de formation
Promotion Promotion récente ou non
ManagerScore Qualité du management
Depart Variable cible : départ ou non

Étape 3 : définition du problème IA

Le modèle doit prédire :

Depart = Oui / Non

Il s’agit donc d’un problème de classification binaire.

Le consultant commence volontairement avec une approche simple :

  • régression logistique ;
  • puis Random Forest.

Dans les RH, l’explicabilité reste essentielle. Les managers doivent comprendre les prédictions générées par l’algorithme.

Étape 4 : préparation des données

Les données RH doivent être nettoyées avant l’apprentissage machine.

  • gestion des valeurs manquantes ;
  • encodage Oui / Non ;
  • normalisation des variables ;
  • contrôle qualité des données.

Étape 5 : analyse exploratoire RH

Avant de lancer les modèles prédictifs, une analyse exploratoire met en évidence plusieurs corrélations fortes.

  • faible satisfaction ;
  • heures supplémentaires élevées ;
  • absence de promotion.

Ces facteurs augmentent significativement le risque de départ.

Étape 6 : construction du modèle prédictif

La régression logistique permet de calculer une probabilité de turnover pour chaque collaborateur.

  • Employé A : 82 % de risque ;
  • Employé B : 11 % ;
  • Employé C : 67 %.
Facteur Impact
Satisfaction faible Très fort
Heures supplémentaires élevées Fort
Salaire faible Fort
Absence de promotion Moyen
Formation faible Modéré

Étape 7 : amélioration avec Random Forest

Le consultant teste ensuite un modèle plus avancé : Random Forest.

Cet algorithme détecte des interactions plus complexes entre les variables RH.

Exemple :

Un salaire faible seul n’explique pas toujours un départ. Mais un salaire faible combiné à un mauvais management et une surcharge de travail augmente fortement le risque.

Résultats du modèle

Modèle Précision
Régression logistique 74 %
Random Forest 86 %

Le cabinet recommande :

  • Random Forest pour la prédiction ;
  • la régression logistique pour l’explication métier.

Traduction opérationnelle RH

L’objectif final est d’aider les équipes RH à agir avant les démissions.

Risque Action RH
> 80 % Entretien manager immédiat
60–80 % Revue salariale
40–60 % Suivi RH renforcé
< 40 % Surveillance normale

Exemple concret de salarié à risque

  • 2 ans d’ancienneté ;
  • salaire sous le marché ;
  • 14 heures supplémentaires par semaine ;
  • satisfaction : 4/10 ;
  • aucune promotion récente.

Le modèle prédit :

Probabilité de départ : 87 %

Actions recommandées :

  • entretien RH ;
  • réduction de charge ;
  • plan d’évolution ;
  • révision salariale.

Exemple Python simplifié 


from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split

X = data[["Salaire","Satisfaction","HeuresSup","Anciennete"]]
y = data["Depart"]

X_train, X_test, y_train, y_test = train_test_split(X, y)

model = RandomForestClassifier()
model.fit(X_train, y_train)

prediction = model.predict(X_test)

  • évolution mensuelle du turnover.

Les limites de l’IA RH

L’intelligence artificielle aide à détecter les risques, mais elle ne remplace jamais le management humain.

Un projet IA RH doit toujours :

  • respecter le RGPD ;
  • éviter les biais algorithmiques ;
  • garder une validation humaine ;
  • expliquer les modèles ;
  • protéger les données collaborateurs.
